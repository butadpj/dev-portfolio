For every type of project you want to do, there are different ways you can setup TypeScript. 
But, regardless of type of project you are building, the requirements for using TypeScript 
is going to be the same:

- A TypeScript compiler (tsc) or other transpilers that convert TypeScript files to JavaScript
  files such as [esbuild (20~30x faster than tsc)](https://esbuild.github.io/),
  [Babel](https://babeljs.io/), or [swc](https://swc.rs/docs/getting-started)
- Installation of <span class="not-prose">`typescript`</span> package for
  type checking _(this will install <span class="not-prose">`tsc`</span> too)_
- [Node.js](https://nodejs.org/en/download/package-manager) for installation of the said packages.

## [How to learn TypeScript?](#how-to-learn-typescript)

You can start by learning TypeScript's basic types (<span class="not-prose">`number`</span>, 
<span class="not-prose">`string`</span>, <span class="not-prose">`boolean`</span>, 
<span class="not-prose">`array`</span>, <span class="not-prose">`object`</span>, 
<span class="not-prose">`null`</span>, <span class="not-prose">`undefined`</span>). You're probably
already familiar with them unless you're completely new to programming. 

Now you can start specifying the types of your **variables**, function **parameters** and **return values**.

<Code
  class="not-prose text-base"
  code={`
    // Syntax #1: colon ":"
    let firstName: string = 'Jane';
    let yearsOld: number = 25;

    // Syntax #2: "as"
    let lastName = 'Oolah' as string;
    let favoriteColors = ["Red", "Blue"] as string[];

    let ids: number[] = [101, 102, 103]; 
    let ids: Array<number> = [101, 102, 103]; // This works too!


    //       typing the parameter   typing the parameter          
    //                 üëá            üëá           üëá  typing the return value  
    function greet(name: string, age: number): string {
        return \`Hello, \${name}!\`;
    }

    //       typing the parameter   typing the parameter          
    //                 üëá            üëá           üëá  typing the return value    
    const greet = (name: string, age: number): string => {
      return \`Hello, \${name}!\`;
    };

    // Typing a more complex data structure (e.g. Objects or array of objects)     
    type User = {
      id: string;
      name: string;
      age: number;
      favoriteColors: string[] // Array<string> works too!
    };

    //  Typing an object
    //        üëá
    const user: User = {
      id: "aBVX26jl2328",
      name: 'Alice',
      age: 22,
      favoriteColors: ["Red", "Black"]
    }

    //  Typing an array of objects. Array<User> works too!
    //        üëá
    const users: User[] = [
      {
        id: "aBVX26jl2328",
        name: 'Alice',
        age: 22,
        favoriteColors: ["Red", "Black"]
      },
      {
        id: "lTOJ51am5113",
        name: 'Paul',
        age: 20,
        favoriteColors: ["Green", "Blue"]
      }
    ]

    //     You can also use the type User here or to other values
    //                    üëá
    function printUser(user: User): string {
      console.log(\`Name: \${person.name}, Age: \${person.age}\`);
    }
  `}
  lang="js"

<div class="text-base">_Not that hard right? You're just typing a colon_ "**:**", or "as", _followed 
by the type you want to specify_</div>

But, uh oh, hold on! ‚úã. This is **not** how you use TypeScript correctly. You're just **unnecessarily** writing 
more codes if you **explicitly declaring** types everywhere üò©. TypeScript is smart enough to figure out the type of the value you just 
declared and carry that type to other places (we call this [type inference](https://medium.com/@sahil90085/typescript-type-inference-a-comprehensive-guide-40c888c6d78f)). 
Now, this is where you should focus when learning TypeScript!


### [Type inference](#type-inference)

_Just remember, you don't need to declare types all the time_ üòä.

<Code
  class="not-prose text-base"
  code={`
    // Example 1: Type inference with primitives
    let name = 'Alice'; 
    let age = 30; 

    // infers 'name' as type 'string'
    console.log(name.toUpperCase()); // Has string methods ‚úÖ

    // infers 'age' as type 'number'
    console.log(age.toFixed(2)); // Has number methods ‚úÖ
    


    // Example 2: Type inference with arrays and objects
    let numbers = [1, 2, 3]; 
    let person = { name: 'Bob', age: 25 };

    console.log(numbers.map(n => n * 2)); // Valid array of numbers ‚úÖ
    console.log(person.name.toUpperCase()); // person.name is a valid string with string methods ‚úÖ   
  `}
  lang="js"
/>

Let's have a more real-world examples üôÇ

<Code
  class="not-prose text-base"
  code={`
    // main.ts

    type User = {
      id: string;
      name: string;
      age: number;
    };

    // Function to fetch users from an API
    const fetchUsers = async () => {
      const response = await fetch(\`/api/users\`);
      if (!response.ok) {
        throw new Error("Failed to fetch users");
      }
      // This is the only time we explicitly type a value
      const userData: User[] = (await response.json()); 
      // const userData = (await response.json()) as User[]; // this works too!    

      return userData; 
    };

    // fetchUsers() returns Promise<User[]> 
    fetchUsers().then((data) => {
      const users = data; // infers 'data' as type 'User[]'

      const usersAge = users.map((user) => user.age);

      console.log(usersAge) // usersAge has a type of number[]
    })
  `}
  lang="js"
/>


## [Conclusion](#conclusion)

You have learned that TypeScript's type system allows you to **detect errors ahead of time** and 
provide **better dev experience**, making it possible for you to make big changes in your codebase
very easily and quickly üëå‚ö°Ô∏è. 

When **setting up TypeScript**, it depends on the project you're building.
You can install the needed packages and configure TypeScript manually or if you're using web frameworks 
(React, Next.js, AstroJS, Angular), they usually have built-in TypeScript support.

When **learning TypeScript**, once you have grasp the basic types, you don't need to explicitly type every
values (variables, parameters, return values). By using type inference, you can just focus on the 
logic and structure of your code rather than writing unnecessary boilerplate codes.
